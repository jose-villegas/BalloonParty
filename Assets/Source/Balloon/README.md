# Balloon

Represents balloons in the game — their state, appearance, spawning, and destruction.

## Folder structure

| Folder | What it owns |
|---|---|
| `Model/` | `IBalloonModel` (read-only interface — extends `IDynamicSlotActor`, `IHitable`, `IHasNudge`), `IWriteableBalloonModel` (mutable — extends `IWriteableDynamicSlotActor`), `BalloonModelBase` (abstract — implements `IWriteableBalloonModel`; constructed via `BalloonModelConfig`; `EvaluateHit` supports `DamageFlags.Piercing` for forced `Pop`), `BalloonModel` (concrete — extends `BalloonModelBase`, also implements `IPaintable`, `IHasDurability`, `IHasScore`, `IHasScoreColor`, and `IHasWriteableItemSlot` — paintable, item-capable, scores in its own color; no attribution when `HitsRemaining > 0` after hit), `ToughBalloonModel` (concrete — extends `BalloonModelBase`, implements `IHasDurability`, `IHasScore`, `IHasScoreColor`; NOT paintable, NOT item-capable; `EvaluateHit` returns `Deflect` on survival; on pop scatters score to random palette colors), `BubbleClusterModel` (concrete — extends `BalloonModelBase`, implements `IHasDurability`, `IHasScoreColor`; no palette color, no item slot; on each hit scatters one score point per damage to random palette colors with `BreaksStreak = true`), `UnbreakableBalloonModel` (concrete — extends `BalloonModelBase`, implements `IHasScore`, `IHasScoreColor`; always `Deflect`; `Piercing` forces `Pop`; on pop scatters score to source color; reads `NudgeOverrides` from config). Item eligibility is determined structurally — `BalloonModel` implements `IHasWriteableItemSlot`; other types do not. Consumers use the narrowest interface: views receive `IBalloonModel`; controllers, spawners, and grid operations use `IWriteableBalloonModel`. `IHasColor` is NOT on `IBalloonModel` — only `BalloonModel` implements it; consumers that need color cast to `IHasColor` |
| `View/` | `BalloonView` — MonoBehaviour implementing `IPoolable` and `ISlotActorView` that binds to a model and renders color, sorting order, and pop VFX. `ActorKind` returns `Dynamic`. Uses `ColorableRendererExtensions.BindColor` to reactively update all `ColorableRenderer[]` children when the model's color changes (guarded by `model is IHasColor` cast). Caches `IBalloonViewBinding[]` and `IBalloonVariant` in `Awake()`. Exposes `Variant` for `BalloonSpawner`. `TweenTracker` and `ItemDisplayService` are `[SerializeField]` references |
| `Controller/` | `BalloonController` — wires model to view and handles hit/deflect/pop routing via `msg.Outcome` switch; `BalloonControllerRegistry` — flat-array model→controller lookup keyed by `IBalloonModel.RegistryHandle`; `HitPipeline` routes each hit through it, and it owns single-pass board-clear teardown; `BalloonControllerContext` — bundles the shared controller dependencies; `BalloonMotionTicker` — central `ITickable` driving every nudge out-and-back with pooled entries (views expose the `IBalloonMotionView` setter surface); `BalloonBalancer` — rebalances the grid after each turn (and, as an `ITickable`, pulses a rebalance at intervals while a projectile is airborne when a move is possible); reserves transit slots in `BalancePathHolder` per-actor, released on animation complete; `PressurePropagation` — directed weighted search behind `BalloonBalancer.TryRelievePressure`: a board-up shove enters at the blocked column's lowest blocker and propagates through occupied movables best-first by cumulative path score (hop alignment minus a heaviness cost derived from `MaxBalanceSteps`), scoring escape slots via the shared `MoveWeightEvaluator`; relocating actors (`PressureResponse.RelocateNearest`/`RelocateFarthest`) are chain terminals, statics block, and the result is a mover-first move list the balancer executes |
| `Spawner/` | `BalloonSpawner` — creates balloon lines at game start and after each projectile death; `BalloonPoolChannel` — pool channel using `InjectingPoolChannel` with `IObjectResolver.Instantiate` |
| `Type/` | `BalloonType` enum (`Simple`, `Tough`, `Unbreakable`, `BubbleCluster`), `IBalloonVariant`, `IBalloonViewBinding`, `ColorableBalloonVariant`, `SimpleBalloonVariant`, `ToughBalloonVariant`, `SoapBubbleClusterVariant`, `UnbreakableBalloonVariant` — per-prefab variant components that initialize type and color on the model. `IBalloonViewBinding` is a secondary interface that any prefab component can implement to receive `Bind(model, disposables)` calls from `BalloonView` — used by `ToughBalloonVariant` to drive shader damage state, by `SoapBubbleClusterVariant` to drive cluster count, time, and rotation, and by `UnbreakableBalloonVariant` to push sphere center, radius, and time to the chrome shader (see `Type/README.md`) |
| `BalloonLifetimeScope` | Legacy scope on balloon prefabs — `autoRun` disabled in Inspector; injection handled by `InjectingPoolChannel` |

`View/` also contains `SpriteColorableRenderer` (`ColorableRenderer<SpriteRenderer>` — sets `renderer.color`), `ParticleColorableRenderer` (`ColorableRenderer<ParticleSystem>` — sets `startColor` and immediately recolors live particles via `Clear`+`Play`), and `IBalloonMotionView` — the setter surface `BalloonMotionTicker` drives (`ApplyNudgePosition`, `CompleteNudge`); the controller owns nudge motion state, the view owns the transform it writes to.

### Rendering and GPU instancing

Balloon materials (`BalloonMaterial`, `SpecularBalloonBlur`, etc.) have GPU instancing enabled. Per-balloon color is applied via `SpriteRenderer.color`, which Unity streams to the shader as `unity_SpriteRendererColorArray` in the instancing buffer. All 6 custom shaders follow the `UnitySprites.cginc` pattern with a `_RendererColor` fallback for non-SpriteRenderer components. Balloon sprites are packed into `Balloons.spriteatlas` for draw call reduction. Drop shadows are pre-baked where used (e.g. the `Balloon` and `Unbreakable` prefabs): `SpriteShadowBaker` (`Shared/Rendering/`) bakes the blurred child-sprite silhouette offline into a `BakedShadow` child sprite, replacing runtime blur-shadow materials. See `Assets/Shaders/BalloonParty/README.md` for the instancing policy and `Assets/Sprites/README.md` for atlas groupings.

## Behaviour

A balloon knows its type, how many hits it can absorb, where it sits in the grid, whether it has settled into position, and whether it carries an item. `BalloonModel` additionally knows its color (`IPaintable`). When any reactive property changes, the view updates automatically via UniRx subscriptions.

`BalloonController` receives its own hits directly: each controller registers with `BalloonControllerRegistry` on spawn, and `HitPipeline` (`Game/`) resolves the owning controller by model and calls `HandleHit` — no per-balloon subscriptions or broadcast filtering. It routes on the pre-computed `msg.Outcome`:

- **`PassThrough`** — balloon survived; projectile continues. `HitsRemaining` was already decremented inside `BalloonModelBase.EvaluateHit`. Crack animation is driven reactively by `HitsRemaining`.
- **`Deflect`** — balloon survived and projectile bounced. Publishes `BalloonDeflectedMessage` and `NudgeMessage(Deflect)`.
- **`Pop`** — balloon destroyed: plays VFX, removes from grid, returns to pool (deferred if the balloon carries an item). Overkill damage is not tracked.
- **`Absorb`** — projectile is destroyed by the actor. `ProjectileView.OnAbsorb` handles the terminal path — publishes `ActorHitMessage(Absorb)` and destroys the projectile. The absorbing actor is not removed from the grid.

`EvaluateHit` on `BalloonModelBase` is **state-mutating** — it decrements `HitsRemaining` (or zeroes it for `Piercing`) and returns the outcome in a single call. `ProjectileView` calls it once and embeds the outcome in `ActorHitMessage`; `BalloonController` reads `msg.Outcome` without calling `EvaluateHit` again.

Neighbor nudges for every hit are handled independently by `NudgeService` in `Nudge/`, which subscribes to `ActorHitMessage` (filtering by `IHasNudge`) and dispatches push-out → return animations to all 6 grid neighbors that also implement `IHasNudge`.

### Pooling

Balloon views are pooled via `PoolManager` / `BalloonPoolChannel`. A `CompositeDisposable` replaces `AddTo(this)` for reactive subscriptions — cleared on each `Bind()` and `OnDespawned()`. External subscribers (e.g. `BalloonController`'s transient item-activation subscription) register via `RegisterDisposeOnDespawn()` so they're cleaned up on pool return. `IBalloonViewBinding` components (e.g. `ToughBalloonVariant`) receive the same `CompositeDisposable` so their subscriptions are cleared automatically in the same lifecycle.

### Animation phases (turn-based)

Balloon animations are organized into sequential, non-overlapping phases per turn:

1. **Hit phase** — projectile bounces and pops or deflects balloons. Each hit nudges neighbors outward and back (handled by `NudgeService`). While the projectile is airborne, `BalloonBalancer.Tick` also pulses a rebalance at `IBalloonsConfiguration.FlightRebalanceInterval` — but only when `HasPossibleMove()` reports the stack can still settle — so a projectile looping wall-to-wall eventually gets a balloon shifted into its path.
2. **Balance phase** — after the projectile dies, a single `Balance()` pass consolidates existing balloons upward before any new lines spawn. Transit slots are reserved per-actor in `BalancePathHolder` and released when the animation completes. `GridBalanceQuery` scores each candidate slot by its support cone, plus a per-actor offset injected via `IBalanceInfluence.WeightBias`. The base implements the entry's *signed* `SeparationBias` (same-type proximity: positive keeps apart — tough; negative clumps together — soap), and `BalloonModel` composes `DiagonalColorBias` on top (candidates score higher per same-color balloon in the diagonal band: the four adjacent diagonals plus the full rows two above/below — own row and the ±1-row outer cells excluded — drifting same-color balloons into diagonal lines). Each pass is a *race round*: unbalanced actors act in `BalancePriority` order (higher intervenes first and wins contested slots; ties keep sweep order), and the entry's `MaxBalanceSteps` caps how many slots an actor rises per **balance window** — the turn's spawn cluster (death-frame + pre/post-spawn runs) shares one budget, and each time-paced in-flight pulse opens a fresh one, so a 1-step heavy still crawls during flight (0 = unlimited) — so heavier types act late and settle slower. Balance requests coalesce: a direct `Balance()` run (the spawner's pre-spawn sweep) services any pending deferred request, so the death-frame publish never duplicates it. Once per turn (the pre-spawn `Balance(relocateRoamers: true)`), a pre-pass runs before the race: the balancer finds every `IPreBalanceRelocatable` actor, offers it the board's resting slots (empty, needing no further settling), and invokes the contract in `BalancePriority` order — each implementer defines its own placement. `UnbreakableBalloonModel` implements it as a jump toward the board's *opposite side*, preferring far slots (squared-distance weighted pick), so it never camps one flank; the race then settles everyone else around the new positions. `TryRelievePressure` is different: it runs `PressurePropagation`, a directed weighted search that bends a board-up shove through occupied movables — each hop scored by its alignment with the incoming shove minus a heaviness cost (`HeavinessPenalty / MaxBalanceSteps`), so chains route around heavy movers when lighter routes exist; the step cap and `BalancePriority` never *gate* under pressure (a shoved balloon moves regardless), heaviness only prices the path. Actors whose entry sets `DirectBalanceMotion` (e.g. the unbreakable) animate straight from where they are to the resolve's *final* slot instead of touring every recorded waypoint — the disturbance wake follows that direct line, matching the visual.
3. **Spawn phase** — new balloon lines spawn with a delay between each line. Spawn uses standalone `DOMove` + `DOScale` tweens (not tracked). `ComputePath` consults the `BalancePathHolder` to detect traversal conflicts with still-animating balance moves. Each wave (a turn's lines, or the initial fill) rolls a per-type spawn quota from the range's count-weight curves (`BalloonTypeWeight.InitialCountWeights` / `WaveCountWeights`, sampled like item counts — chance of 0, 1, n; empty curve = uncapped) alongside the concurrent `MaxCountOverride`, then pre-picks its whole batch of entries and deals them lightest-first by the catalog entry's `SpawnWeight` (the type's physical spawn weight) — successive lines land progressively lower, and within each line the columns are processed shallowest-first (`ProbeEntryRow` depth order, random tie-break), so the heaviest types (e.g. tough) always take the lowest available slots. Pure ordering: the wave's type mix is untouched. Active counts are reserved at batch time and released for entries the wave never places.
4. **Post-spawn balance** — a `BalanceBalloonsMessage` triggers a deferred balance pass (next frame) to settle any newly created gaps.

Additionally, a **direct projectile pop** (`DamageFlags.DirectHit` — never AOE items) can earn one extra balloon, spawned immediately: each pop rolls `IBalloonsConfiguration.PopSpawnChance` (global, not per level; 0 = off). The loose spawn takes its column's own entry only — no rehoming, no pressure, no overflow charge, columns without room skipped — and is excluded from item checks. Each spawn is budget-gated: the automated spawns stop at grid capacity minus `IBalloonsConfiguration.PopSpawnFreeRows` × columns (headroom further clamped to one turn-wave's worth, the range's `SpawnLines` × columns), so they can never pack the play space solid.

### Motion arbitration

Position motion has two drivers: DOTween (spawn and balance paths, sequenced by the `TweenTracker` component on each view) and `BalloonMotionTicker` (nudge out-and-back). A ticker-driven nudge escapes `transform.DOKill()`, so each driver cancels the other explicitly when it takes over:

- **Nudge** (`BalloonView.Nudge`) kills standalone spawn tweens via `transform.DOKill()`, then calls `BalloonMotionTicker.StartNudge` — starting a nudge for a view silently replaces its running one. If the balloon was mid-scale-up, a parallel `DOScale` smoothly finishes scaling to full size.
- **Balance** (`BalloonBalancer.AnimatePaths`) calls `TweenTracker.Kill()` + `transform.DOKill()` + `BalloonMotionTicker.CancelNudge()`, then appends its `DOPath` via `tracker.Append(tween)`.
- **Despawn** calls `tracker.Kill()` + `transform.DOKill()` + `CancelNudge()` — cleans up everything.

### Scale recovery

When nudge or balance interrupts a spawning balloon, `transform.DOKill()` kills the standalone scale tween mid-animation. Both nudge and balance capture the current scale beforehand and create a parallel `DOScale(Vector3.one, duration)` if the balloon hasn't reached full size — so the balloon smoothly finishes scaling alongside its movement.

## Interactions

- **SlotGrid** — balloons occupy positions in it; stores models and views in parallel arrays. Spawner places, controller removes, balancer relocates. `_grid.ViewAt(slotIndex)` reaches views from models.
- **BalloonBalancer** — moves balloons when gaps appear; runs once before spawning (pre-spawn consolidation), once after (deferred via `BalanceBalloonsMessage`), and at `FlightRebalanceInterval` intervals while a projectile is airborne (gated on `HasPossibleMove()`). Reserves transit slots in `BalancePathHolder` per-actor; releases on animation complete
- **NudgeService** — subscribes to `ActorHitMessage` and `NudgeMessage`; nudges neighboring and deflecting balloon views (executed by `BalloonMotionTicker` via `BalloonView.Nudge`)
- **ProjectileView** — calls `EvaluateHit(new DamageContext(1))` on collision, embeds pre-computed outcome in `ActorHitMessage`
- **ScoreController** — on each `ActorHitMessage` with `Outcome == Pop` or `PassThrough`, casts actor to `IHasScoreColor` and calls `ResolveScoreAttribution(msg.Context, attributions)`. All returned `ScoreAttribution` entries are published together as one scatter group — a single `GroupSize` covers every point from the pop so the UI fans them out together.
- **PoolManager** — `BalloonView` instances are pooled via `BalloonPoolChannel`; pop VFX pooled via `ParticlePoolChannel`
- **TweenTracker** — generic `MonoBehaviour` in `Shared/` that manages tween sequencing (append, replace, kill)
- **Item system** — balloons are one host for the game-wide item system (in `Item/`). `BalloonView` calls `ItemDisplayService.Bind()`/`Unbind()` to connect item visuals; the item system itself has no knowledge of balloons
- **BalloonsConfiguration** — balloon prefab entries, spawn animation timing, balance timing, nudge defaults, pop VFX
- **GamePalette** — resolves color name → `UnityEngine.Color` for renderer tinting
- **DisturbanceFieldService** — `BalloonController` stamps on pop (`BalloonPop` profile), `BalloonBalancer` stamps during balance path (`BalloonPath` profile), `BalloonSpawner` stamps during spawn path (`BalloonPath` profile), and `BalloonController.Deflect` stamps at the balloon's slot when a projectile deflects off it and its entry sets `DeflectStampScale` (`BalloonDeflect` profile × the scale — unbreakable wide, tough slot-sized; pops already stamp via `BalloonPop`), and `ToughBalloonVariant` pulses a warning stamp while a tough sits at 1 hit left — the reserved Tough palette color while **alternating** a repel/attract pulse each tick (pushing the field out, then pulling it back in — a "breathing" halo) at the `ToughWarning` profile's `Interval` (the profile supplies cadence + magnitude + radius; the variant flips the sign; 0 interval = off). `RainbowBalloonVariant` does the same constantly (not gated on HP) — a single **color-only** stamp (`RainbowColor`, authored `Strength = 0` so it applies no force, R stays at rest) that tags nearby specks with the *next* of the level's allowed colors each pulse, so they cycle the palette. The profile's `Interval` paces how fast the rainbow cycles colors. `UnbreakableBalloonVariant` runs a periodic **burst** (`UnbreakableBurst`), tagged its own reserved `Unbreakable` palette color, that shoves the field's clouds out and, at the same point, enables a matching puff of specks by publishing a `SpeckSpawnRequestMessage` (the generalized "spawn N specks here" request the `SpeckField` resolves against its `SpeckProfile` for that `SpeckSource`). Both the cadence and the force come from the profile: `StampProfile.Interval` is the seconds between emissions (0 = not a periodic emitter). All these constant emitters pass `reportImpact: false` so their steady pulses don't rustle bushes. The field's R channel is **signed density** around a 0.5 rest: a stamp's `Strength > 0` bumps R up (repulsion — specks pushed away, the default for pops/bombs/etc.), `Strength < 0` digs R down (attraction — specks pulled in), and `Strength = 0` writes only the A-channel color tag (pure color, no force, no bush rustle). Specks flow down the R gradient; a color tag lights a speck by the tag's remaining life independent of agitation — so color and force are separable — all via `Stamp()` using `StampProfile` from `DisturbanceFieldSettings`. Creates visible disturbances in Puff clouds
