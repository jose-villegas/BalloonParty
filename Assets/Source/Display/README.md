# Display

Camera and screen-adaptation logic.

## Contents

| File | What it provides |
|---|---|
| `OrthogonalSizeCameraController` | MonoBehaviour — sets the camera's orthographic size based on the device's aspect ratio. Reads `GameDisplayConfiguration` (injected) which maps aspect ratios to optimal orthographic sizes. Has a serialized `Camera` reference to avoid `Camera.main` lookup issues during additive scene loading. When `_continuous` is enabled, updates every `LateUpdate` to react to resolution or orientation changes; otherwise applies once on `Start` |
| `CameraShakeService` | MonoBehaviour — punches the gameplay camera on every `SpawnBlockedMessage` (a rejected balloon). The shake is an **additive offset** applied as a per-frame delta in `LateUpdate`, so it composes with whatever else drives the camera (the heart-drain pan just absorbs it) instead of fighting over the absolute position. Runs unscaled so slow-mo can't stretch it; skipped only while the current cinematic declares `CinematicTraits.BlocksShake` (the level-up states). Serialized `Camera` reference plus inspector-tunable duration / strength / vibrato. Registered via `RegisterComponentInHierarchy<CameraShakeService>()`; place it on the camera GameObject |
| `SceneCaptureService` | MonoBehaviour on the main camera — a **shared low-res capture of the scene's visuals**, bound globally as `_SceneCaptureTex` (divisor + every-Nth-frame interval from `IGameDisplayConfiguration`). Any effect that wants "what the screen roughly looks like" acquires it instead of using a `GrabPass` (a full-screen mid-frame resolve on tile GPUs); the first consumer is the Unbreakable's chrome reflection (`UnbreakableBalloonVariant` ref-counts via `Acquire`/`Release` on enable/disable). Renders only while someone holds an acquire; skipped frames keep the previous capture. The serialized layer mask picks what gets captured — consumers sampling it should exclude their own layer to avoid feedback. The capture camera clears with **alpha 0**, so the RT's alpha channel doubles as a sprite-coverage mask (`ScreenSpaceLightService`'s occlusion source). One shared mask/resolution on purpose; a second consumer with different needs is the moment to generalize. Its inspector live-previews the capture RT in play mode (`SceneCaptureServiceEditor`). Registered via `RegisterComponentInHierarchy<SceneCaptureService>()`; **must be placed on the main camera in the Game scene** (scope build fails if absent) |
| `ScreenSpaceLightService` | MonoBehaviour on the main camera, next to `SceneCaptureService` (**prototype** — see `PLAN-ScreenSpaceLight.md`). Fakes a global 2D light: smears the capture toward the light into a tiny buffer (shadow in A, color bounce in RGB) via three low-res blits — directional 8-tap smear, 3×3 box soften, then a temporal EMA blend against a ping-pong history buffer (`_temporalResponse`) to kill capture-resolution texel flicker — then composites it over the whole frame as a camera-fitted quad with a multiplicative blend — no post-processing readback, no changes to other materials. The smear/overlay `Shader` references are **serialized** (assign them — builds strip shaders only referenced by name; `Shader.Find` is an editor-only fallback). The overlay quad lives on `TransparentFX` (keep that layer OUT of the capture mask — that's what makes frame-to-frame feedback impossible) and sorts on `Sky`/32000, above gameplay and below UI. All knobs serialized on the component, live-tunable in play mode; disable the component to A/B. Its inspector live-previews the light buffer in play mode (bounce RGB + shadow alpha, `ScreenSpaceLightServiceEditor`). Not DI-registered — plain component, no dependencies beyond its sibling capture service |

## How it works

`GameDisplayConfiguration` (in `Configuration/`) holds a reference world width and height. `GetOrthogonalSize()` computes the orthographic size that fits both against the current screen aspect (`max` of fit-width and fit-height), so the play field is fully visible on every device without letterboxing or clipping. Its custom editor previews the resulting size for common device ratios, in the inspector and as a Scene-view overlay.

The controller is registered via `RegisterComponentInHierarchy<OrthogonalSizeCameraController>()` in `GameLifetimeScope` and `LaunchLifetimeScope`. Place it on the camera GameObject in each scene with the camera reference wired in the Inspector.
